# SAP Application Frontend - Deploy Template

A ready-to-use template for deploying static web apps to [SAP Application Frontend](https://help.sap.com/docs/application-frontend) using the `afctl` CLI, with a GitHub Actions pipeline included.

## Quick Start

### 1. Use this template

Click **Use this template** on GitHub, or clone and re-init:

```bash
git clone https://github.com/SamS-Lemongrass/appfront-template.git my-app
cd my-app
rm -rf .git && git init
```

### 2. Add your app

Put your static files in the `webapp/` folder. The template includes a sample `index.html` to get started.

For framework-based apps (React, Vue, Angular, etc.), put your source in the project root, build to `dist/`, and update `APP_BUILD_PATH` in the pipeline config.

### 3. Deploy manually (optional)

```bash
npm install -g @sap/appfront-cli
afctl login --service-key <path-to-service-key.json> -a <api-url>
afctl push webapp --activate
```

### 4. Set up automated deployment

See [Pipeline Setup](#pipeline-setup) below.

## Pipeline Setup

The included GitHub Actions pipeline (`.github/workflows/deploy.yml`) deploys to Application Frontend on every push to `main`.

### Prerequisites

1. **SAP BTP subaccount** with Application Frontend service subscribed
2. **Service key** for your existing `app-front` service instance

### Get your service key

```bash
cf login -a https://api.cf.eu10-005.hana.ondemand.com --sso

# List service keys for your app-front instance
cf service-keys <your-appfront-instance>

# View the key (copy the JSON output)
cf service-key <your-appfront-instance> <key-name>
```

### Configure GitHub Secrets & Variables

Go to your repo **Settings > Secrets and variables > Actions**:

| Type | Name | Value |
|------|------|-------|
| **Secret** | `APPFRONT_SERVICE_KEY` | Full JSON from `cf service-key` output |
| **Variable** | `APPFRONT_API_URL` | `https://api.eu10.dt.appfront.cloud.sap` |

**Optional variables:**

| Variable | Default | Description |
|----------|---------|-------------|
| `APP_BUILD_PATH` | `webapp` | Folder to deploy (use `dist` for framework builds) |
| `APP_NAME` | repo name | Application name in AppFront |
| `APP_VERSION` | `v1` | Version label |
| `BUILD_COMMAND` | _(none)_ | Build command, e.g. `npm run build` |
| `NODE_VERSION` | `22` | Node.js version for the runner |

### Pipeline Behaviour

| Event | Action |
|-------|--------|
| Push to `main` | Build (if configured) and deploy to AppFront |
| Pull request to `main` | Build validation only (no deploy) |
| Manual trigger | Deploy on demand via Actions tab |

## Common Scenarios

### Plain HTML / static site (default)

No extra config needed. Put files in `webapp/`, push to `main`.

### Vite / React / Vue / Angular

1. Set up your framework project in the repo root
2. Set these GitHub variables:
   - `BUILD_COMMAND` = `npm run build`
   - `APP_BUILD_PATH` = `dist`

### SAPUI5 / Fiori

1. Put your UI5 project in the repo root
2. Set these GitHub variables:
   - `BUILD_COMMAND` = `npm run build`
   - `APP_BUILD_PATH` = `dist` (or `webapp` if no build step)

### Next.js (static export)

1. Configure `output: 'export'` in `next.config.js`
2. Set these GitHub variables:
   - `BUILD_COMMAND` = `npm run build`
   - `APP_BUILD_PATH` = `out`
